Что выведет программа? Объяснить вывод программы.
Объяснить как работают defer’ы и их порядок вызовов.

```go
package main

import "fmt"

func test() (x int) {
	defer func() {
		x++
	}()
	x = 1
	return
}

func anotherTest() int {
	var x int
	defer func() {
		x++
	}()
	x = 1
	return x
}

func main() {
	fmt.Println(test())
	fmt.Println(anotherTest())
}
```

Ответ:
```text
Программа выведет
2
1

В сигнатуре функции test() указан именованный возвращаемый параметр х.
Именованные параметры имеют следующие особенности:
    1. Они инициализируются нулевыми значениями в момент вызова функции
    2. Функции/методы, вызываемые в defer могут изменять их значения после вызова return

Объяснить вторую особенность можно следующим образом.
Во-первых, вызов return не атомарен, его можно разделить на два действия:
    1. Возвращаемое значение сохраняется на стеке
    -
    2. Функция передаёт управление "родительской" функции
Во-вторых, нужно понимать, что функции, вызванные в defer, выполняются между
первым и вторым действиями, указанными выше:
сразу после того, как значение (фактическое значение(return 1) или значение
некоторой переменной(return var)), указанное в return, будет сохранено на стеке
и
до того, как функция вернёт управление.
То есть в том месте, где я поставил прочерк.

В случае функции test() именно значение переменной 'х' будет сохранено на стеке.
Анонимный метод, вызываемый в defer, присваивание x=1 и return x - все эти действия
производятся с одной и той же переменной.

В случае функции anotherTest() поведение отличается.
Присваивание х=1 и инкрементирование х++ относятся к одной и той же переменной, как и в test().
Отличие заключается в том, что происходит при вызове return x.
При вызове return x мы сохраняем на стеке значение переменной х(==копируем значение х в другую переменную).
Инкрементирование значения переменной х в defer не влияет на значение, сохранённое на стеке.

Вызываемые в defer функции/методы выплняются после вызова return или panic,
но до реального возврата вызывающей функции в порядке LIFO.
```
